# LLM Fundamentals (Engineer's Lens)

নিচে টোকেন, টোকেনাইজেশন, কনটেক্সট উইন্ডো এবং কস্ট ম্যাপিং সম্পর্কে সহজ ভাষায় আলোচনা করা হলো:

## ১. Tokens (টোকেন)
LLM সরাসরি শব্দ বা বাক্য বোঝে না, এরা বোঝে সংখ্যা। **টোকেন হলো টেক্সটের সবচেয়ে ছোট ইউনিট** যা মডেল প্রসেস করতে পারে। 
* **ইংরেজিতে:** ১টি টোকেন সাধারণত ৪টি ক্যারেক্টার বা ০.৭৫টি শব্দের সমান। (যেমন: `"Hello world"` = ২টি টোকেন)।
* **বাংলা/অন্য ভাষায়:** ১টি শব্দ ভেঙে ২-৩টি টোকেন হতে পারে, কারণ ইংরেজির তুলনায় অন্য ভাষার জন্য ভোকাবুলারি ছোট থাকে।
* **কেন গুরুত্বপূর্ণ:** মডেলের মেমোরি এবং খরচ সবকিছুই টোকেনের ওপর ভিত্তি করে হিসাব করা হয়।

## ২. Tokenization (টোকেনাইজেশন)
টেক্সটকে টোকেনে ভেঙে ফেলার প্রক্রিয়াকে **টোকেনাইজেশন** বলে। 
* **কীভাবে কাজ করে:** এটি মূলত একটি ডিকশনারি (Vocabulary) ব্যবহার করে টেক্সটকে সংখ্যায় (Integer IDs) কনভার্ট করে। 
* **Engineer's Note:** জনপ্রিয় অ্যালগরিদম হলো **BPE (Byte-Pair Encoding)** বা **WordPiece**। এটি পুরো শব্দ না ভেঙে **Sub-word** লেভেলে ভাঙে। ফলে নতুন বা বিরল শব্দ (Out-of-Vocabulary words) হ্যান্ডেল করা সহজ হয়।
* **উদাহরণ:** `"unbelievable"` শব্দটি ভেঙে `"un"` এবং `"believable"`— দুটি টোকেন হতে পারে।

## ৩. Context Window (কনটেক্সট উইন্ডো)
এটি হলো মডেলের **"মেমোরি লিমিট"**। একটি নির্দিষ্ট সময়ে মডেল সর্বোচ্চ কতগুলো টোকেন প্রসেস করতে পারে (Input + Output মিলিয়ে), তাকে কনটেক্সট উইন্ডো বলে।
* **উদাহরণ:** 8K, 32K, 128K, বা 1M Tokens.
* **কেন গুরুত্বপূর্ণ:** আপনি যদি 128K কনটেক্সট উইন্ডোর মডেলে 130K টোকেনের ডকুমেন্ট দেন, তবে মডেল সেটি পুরোপুরি পড়তে পারবে না বা এরর দেখাবে। 
* **Engineer's Note:** কনটেক্সট উইন্ডো যত বড় হয়, মডেলের "Attention Mechanism" এর কারণে কম্পিউটেশনাল খরচ (Time & Memory) স্কোয়ারলি (Squarely) বাড়তে পারে।

## ৪. Mapping to Cost (খরচের হিসাব)
LLM API-গুলোর (যেমন OpenAI, Anthropic) মূল্য নির্ধারিত হয় টোকেন সংখ্যার ওপর ভিত্তি করে। সাধারণত প্রতি **1 Million (1M) টোকেনের** জন্য চার্জ করা হয়।

খরচ মূলত ২ ভাগে ভাগ হয়:
1. **Input Tokens (Prompt):** আপনি মডেলকে যা পাঠাচ্ছেন (System prompt + User query + Context)।
2. **Output Tokens (Completion):** মডেল আপনাকে যা রিপ্লাই দিচ্ছে।

**গুরুত্বপূর্ণ বিষয়:**
* **Output টোকেনের দাম বেশি:** আউটপুট টোকেন জেনারেট করতে মডেলকে একে একে হিসাব করতে হয়, তাই ইনপুট টোকেনের তুলনায় আউটপুট টোকেনের দাম ২ থেকে ৪ গুণ বেশি হয়।
* **Cache Discount:** অনেক API (যেমন OpenAI Prompt Caching) একই ইনপুট বারবার পাঠালে ইনপুট টোকেনের দামে ৫০-৯০% ছাড় দেয়।
* **Cost Optimization:** ইঞ্জিনিয়ার হিসেবে আপনার কাজ হলো সিস্টেম প্রম্পট ছোট রাখা, অপ্রয়োজনীয় কনটেক্সট বাদ দেওয়া এবং ছোট/সস্তা মডেল ব্যবহার করা যেখানে সম্ভব।
